Configuration

NOMIK relies on environment variables for its configuration, ensuring flexibility and portability across different deployment environments. This approach allows you to manage settings without modifying the codebase, making it easy to adapt NOMIK to various project contexts and operational requirements.

This page details the environment variables recognized by NOMIK, their purpose, and how to set them up.

Environment Variables

The following table lists the environment variables used by NOMIK, providing a description, the functions that utilize them, and whether they are required for the application to run correctly.

Name Description Used By Required?
NOMIK_PROJECT_ID A unique identifier for the NOMIK project. This ID is crucial for distinguishing data and operations between different projects utilizing the NOMIK system. getProjectId Yes
NOMIK_ROLE Defines the operational role or access level for the current NOMIK instance. This can influence available features or permissions. getRole Yes
NOMIK_SAMPLING Controls whether specific data sampling features are enabled or disabled within NOMIK. Set to true or false. isSamplingEnabled No

Setting Up Configuration

You can configure NOMIK by setting the necessary environment variables in your operating system or deployment environment. The method varies depending on your setup.

Local Development

For local development, you can set environment variables directly in your shell or use a .env file if your project utilizes a tool like dotenv.

  • Shell (Bash/Zsh):

    Set variables temporarily for your current session:

    export NOMIK_PROJECT_ID="your_project_id_here"
export NOMIK_ROLE="developer"
export NOMIK_SAMPLING="true"
npm start

    For persistent settings, add these lines to your shell's configuration file (e.g., ~/.bashrc, ~/.zshrc) and then `source` the file.

  • Windows Command Prompt: 
    set NOMIK_PROJECT_ID="your_project_id_here"
set NOMIK_ROLE="developer"
set NOMIK_SAMPLING="true"
npm start
  • PowerShell: 
    $env:NOMIK_PROJECT_ID="your_project_id_here"
$env:NOMIK_ROLE="developer"
$env:NOMIK_SAMPLING="true"
npm start
  • .env File:

    Create a file named .env in the root of your project and add the variables:

    NOMIK_PROJECT_ID="your_project_id_here"
NOMIK_ROLE="developer"
NOMIK_SAMPLING="true"

    Ensure your application's startup script or environment handles loading variables from this file.

Server Environments

In server environments (e.g., production, staging), you should configure environment variables through your hosting provider's interface or orchestration tool.

  • Containerization (Docker, Kubernetes):

    Specify environment variables in your Dockerfile, docker-compose.yml, or Kubernetes deployment manifests. For sensitive information, use Docker secrets or Kubernetes secrets.

    Example docker-compose.yml snippet:

    services:
  nomik-app:
    image: nomik/app:latest
    environment:
      - NOMIK_PROJECT_ID=your_production_project_id
      - NOMIK_ROLE=production
      - NOMIK_SAMPLING=false
  • PaaS Providers (Heroku, Vercel, Netlify, AWS Amplify):

    Most Platform-as-a-Service (PaaS) providers offer a dedicated section in their dashboard to manage environment variables for your deployed applications.

  • Cloud VMs (AWS EC2, Google Cloud Compute Engine, Azure VMs):

    You can set environment variables in system-wide configuration files (e.g., /etc/environment, /etc/profile), in a service manager configuration (e.g., systemd unit files), or within your deployment scripts.
