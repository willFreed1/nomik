Overview

NOMIK is a project analysis toolkit designed to process source code and represent its structure, dependencies, and relationships as a navigable graph. It helps developers understand large codebases by transforming static code into a dynamic, queryable graph database. The system primarily functions as a command-line interface (CLI) tool that parses project files, extracts relevant information, and persists it into a graph database like Neo4j.

How It Works

The NOMIK system operates by transforming source code into a structured graph representation. This process typically begins when a user invokes the command-line interface (CLI) to analyze a project. The cli module orchestrates the workflow, reading project configuration via the core module, which handles environment variables and validation.

Once configured, the cli module delegates the parsing task to the parser module. The parser module traverses the project's source files, extracting abstract syntax tree (AST) information, function calls, variable usages, and other code elements. It generates unique node IDs for these elements, preparing them for graph representation.

The extracted data is then passed to the graph module. This module is responsible for connecting to the graph database (e.g., Neo4j, as indicated by neo4j.driver.ts), creating nodes and relationships based on the parsed data, and persisting them. The graph module encapsulates the database interactions, allowing for a decoupled data storage layer.

Throughout this process, the core module provides fundamental utilities such as configuration loading, validation, and logging services, ensuring consistent behavior across all other modules.

Key Modules

  • core: Provides foundational functionalities including configuration management, logging, and common utilities used throughout the system. It ensures consistent behavior and setup across different parts of NOMIK.
  • cli: The command-line interface module, responsible for user interaction, command parsing, and orchestrating the analysis workflow. It reads project configurations and initiates the parsing and graph generation process.
  • parser: This module is dedicated to static analysis of source code. It extracts structural information, identifies function calls, and generates raw data that can be transformed into a graph representation. Functions like createNodeId and findEnclosingFunctionName are central to its operation.
  • graph: Manages the interaction with the graph database. It's responsible for establishing connections (e.g., to Neo4j), translating parsed data into graph nodes and edges, and persisting them. Functions like connect, disconnect, and createGraphService are key.
