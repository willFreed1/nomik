# @genome/graph

The persistence layer managing interactions with the graph database (Neo4j).

## Features

- **Schema Management**: Auto-initializes constraints (uniqueness) and indexes (performance) on startup.
- **Driver Abstraction**: Wraps `neo4j-driver` to provide a simpler, type-safe API.
- **Query Modules**:
  - `write.ts`: Upsert nodes and create edges using parameterized Cypher.
  - `read.ts`: Impact analysis, dependency chain traversal, dead code detection.

## Key Queries

### Impact Analysis
Uses `apoc.path.subgraphNodes` to traverse `DEPENDS_ON`, `CALLS`, and `imports` relationships up to a configurable depth (default: 5).

### Upsert Pattern
Uses `MERGE` to ensure idempotency. Nodes are identified by a deterministic hash ID generated by the parser.

## Configuration
Requires a running Neo4j instance.
- **Env Vars**: `NEO4J_URI`, `NEO4J_USER`, `NEO4J_PASSWORD`.
